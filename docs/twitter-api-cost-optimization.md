# Twitter API コスト最適化ガイド

## Twitter APIの価格

- **Freeプラン**: 無料だが、ツイート取得は不可（投稿のみ）
- **Basicプラン**: 月額$100（約15,000円）- 月間10,000件のツイート取得
- **Proプラン**: 月額$5,000（約75万円）- 月間1,000,000件のツイート取得
- **Enterpriseプラン**: 月額$42,000〜$210,000（約560万円〜2,800万円）

## コスト削減戦略

### 1. 監視対象アカウントを最小限に

現在、**最重要アカウント3つのみ**を監視：
- `@OpenAI` - 公式アカウント
- `@anthropicai` - Anthropic公式
- `@sama` - Sam Altman（速報情報）

これにより、APIリクエスト数を大幅に削減できます。

### 2. RSSフィードを優先的に使用

公式ブログのRSSフィードは**完全無料**で、以下の情報を取得できます：
- OpenAI Blog
- Anthropic Blog
- Google DeepMind
- xAI Blog
- Microsoft AI Blog

Twitter APIを使わずに、これらのRSSフィードから主要な情報を取得できます。

### 3. コミュニティソースを活用

以下の無料ソースからも情報を取得できます：
- **Reddit**: MachineLearning, LocalLLaMA, OpenAI などのサブレディット
- **HackerNews**: AI/ML関連の投稿
- **HuggingFace**: ディスカッション
- **GitHub Issues**: オープンソースプロジェクトの議論

### 4. リクエスト間隔の調整

- 現在: リクエスト間に3秒待機
- これにより、レート制限を回避しつつ、コストを抑えられます

### 5. エンゲージメントフィルタリング

- 高優先度アカウント: エンゲージメント10以上
- これにより、重要な投稿のみを取得し、APIリクエスト数を削減

## 推奨される運用方法

### 無料で運用する場合

1. **Twitter APIは使わない**
   - RSSフィードとコミュニティソースのみを使用
   - 十分な情報が取得できます

2. **監視対象を最小限に**
   - 最重要アカウント3つのみを監視
   - Basicプラン（$100/月）で運用可能

### Basicプラン（$100/月）で運用する場合

- 監視対象: 3〜5アカウント
- リクエスト間隔: 3秒
- エンゲージメントフィルタ: 10以上
- これにより、月間10,000件の制限内で運用可能

## 代替案

### 1. RSSフィードを拡張

公式ブログ以外にも、以下のRSSフィードを追加可能：
- 技術ブログ
- ニュースサイト
- コミュニティブログ

### 2. スクレイピング（⚠️ 推奨されません）

- **利用規約違反**: X（Twitter）の利用規約でスクレイピングは明確に禁止
- **法的リスク**: アカウント凍結や訴訟のリスク（実際に1億円以上の損害賠償請求事例あり）
- **技術的困難**: ログイン必要、CAPTCHA、レート制限など
- **詳細**: `docs/twitter-scraping-risks.md` を参照

### 3. 手動チェック

- 重要なアカウントは手動でチェック
- システムは補助的な役割に

## 結論

**個人利用の場合、Twitter APIは必須ではありません。**

RSSフィードとコミュニティソース（Reddit、HN）で十分な情報が取得できます。
Twitter APIは「補完的な情報源」として、必要に応じて最小限のアカウントのみを監視することを推奨します。
